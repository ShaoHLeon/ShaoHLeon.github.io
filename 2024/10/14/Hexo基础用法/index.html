Hexo基础用法

Created2024-10-14|Updated2024-10-15|Blog
|Word Count:529|Reading Time:1mins|Post Views:

Hexo 是一个快速、简洁且强大的博客框架。它基于 Node.js，使用 Markdown 语法编写文章，并可以通过多种主题、插件进行自定义。

一、写作

创建文章

[layout]:

  • post 文章(default) 在source/_posts
  • page 页面 在source
  • draft 草稿 在source/_drafts

可以通过编辑 _config.yml 文件中的 default_layout 参数来设置默认布局。

文章名字格式

可以编辑 _config.yml 文件中的 new_post_name 参数来设置，默认为 :title.md。

Placeholder 描述
:title 标题（小写，空格将会被替换为短杠）
:year 建立的年份，比如，2015
:month 建立的月份（有前导零），比如，04
:i_month 建立的月份（无前导零），比如，4
:day 建立的日期（有前导零），比如，07
:i_day 建立的日期（无前导零），比如，7

草稿draft

~/blog$ hexo new draft <title> 创建草稿文章到 source/_drafts 文件夹内，可以使用 ~/blog$ hexo publish [layout] <title> 命令将草稿文章移动到想要的layout中。

默认情况下不显示草稿。您可以在运行Hexo时添加--draft选项，或者启用 _config. yml 中的render_drafts设置来渲染草稿。

<h3 id="Scaffolds-模板"><a href="#Scaffolds-模板" class="headerlink" title="Scaffolds(模板)"></a>Scaffolds(模板)</h3><p><strong>Scaffolds</strong>（模板）是用于定义新建页面、文章或草稿的默认结构的文件。当你使用 <code>hexo new</code> 命令创建新文章或页面时，Hexo 会根据 Scaffolds 中的模板生成相应的 Markdown 文件。这些模板包含了文章的基础结构，例如 Front-matter（YAML 头部信息）部分。</p> <p>Scaffolds 模板文件位于 Hexo 项目的 <code>scaffolds</code> 目录中。默认情况下，Hexo 会提供以下几种模板：</p> <ul> <li><code>post.md</code>：用于创建新文章的模板。</li> <li><code>page.md</code>：用于创建新页面的模板。</li> <li><code>draft.md</code>：用于创建草稿的模板。</li> </ul> <p><img
